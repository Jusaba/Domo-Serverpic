Recoge el texto recibido del servidor lo descompone en remitente y orden.

Comrpueba si se ha recibido un menaje del servidor, si se recibe y es una solicitud de servicio ( empieza por # ), comprueba si la peticion corresponde con un comando de servicio, si es asi, actua y pone el flag de recepcion de mensaje a 0 para que la funcion que la llamo no tenga que analizar la orden, es decir, la funcion que llamo a esta, si el mensaje es de servivio no actuara ( como si no se hubiera recibido mensaje ) y si no es de servicio, esta funciÃ³n le devolverÃ¡ una estructura Telegrama con el remitente y el mensaje.

Lar ordenes de servicio son:

  #factory.-              Borra la flash y deja el modulo 'virgen'
  #nombressid.-           Cambia en EEPROM el nombre de la SSID actual por la que se le envia como parametro en esta orden
  #passwordssid.-         Cambia en EEPROM el password de la SSID actual por el que se le pasa como parametro en esta orden
  #usuarioservidor.-      Cambia en EEPROM el nombre de usuario del servidor actual por el que se le pasa como parametro en esta orden
  #websocket.-            
                          #webSocket.- Devuelve el estado del flag lWebSocket
                          #websocket-:-lEstado Pone el flag lWebSocket a lEstado
  #homekit.-              
                          #homekit.- Devuelve el estado del flag lHomeKit
                          #homekit-:-lEstado Pone el flag lHomeKit a lEstado
  #estadisticas.-              
                          #estadisticas.- Devuelve el estado del flag lestadisticas
                          #estadisticas-:-lEstado Pone el flag lestadisticas a lEstado

  #ESPInfo.-              Devuelve una cadena con la informaciÃ³n del chip delmodulo
